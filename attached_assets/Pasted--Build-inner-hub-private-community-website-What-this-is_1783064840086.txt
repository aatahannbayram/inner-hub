# Build: inner.hub — private community website

## What this is
A restrained, editorial, invitation-only website for **inner.hub** — a private community for founders, investors, and researchers, based in İstanbul. The site's job is not to sell; it is to **signal**. It should feel quiet, confident, and exclusive — the kind of page where the restraint itself is the message. Reference points for tone: the calm of Are.na, the editorial minimalism of early NTS Radio, the composed typography of Anthropic's brand.

The public site is intentionally minimal. One long scrolling page. The only interactive element is a quiet "request an invitation" form.

---

## Tech stack (use exactly this)
- **Next.js (App Router) + TypeScript**
- **Tailwind CSS** for styling
- **Server-rendered content.** Every word of copy must be present in the server-rendered HTML — no client-only rendering of text content. (This site replaces a JavaScript-shell build that was invisible to search engines; do not repeat that mistake.)
- A **server-side API route** for the invitation form
- **Database**: Replit's built-in Postgres (or SQLite if simpler) to store invitation requests
- No heavy UI libraries. Minimal dependencies. Clean, commented code.

## Brand system — "Quiet Signal" (follow precisely)

### Colors (define as Tailwind theme tokens / CSS variables)
- **Ink** `#0A0A0A` — primary text; dark surfaces
- **Bone** `#F4F1EC` — primary background / paper
- **Inner Green** `#18FF85` — the single accent. Used sparingly and precisely. It is a signal, never decoration.

Default: **Bone background, Ink text.** Green appears only in the brand mark, the section-index accents, and one or two deliberate moments.

Detail: set the browser text-selection color to Inner Green background with Ink text.

### Typography
- **Fraunces** (serif, Google Fonts) — display and headlines. Large statements are set in **Fraunces italic**.
- **Geist** (sans, via the `geist` npm package) — body and UI. Fallback if unavailable: **Inter Tight** (Google Fonts).
- **JetBrains Mono** (Google Fonts) — labels, section numbers, metadata, form labels. Always uppercase, small size, generous letter-spacing (~0.08–0.12em).

Load all fonts with `next/font` (no layout shift, no FOUT). Type scale: hero headline `clamp(2.5rem, 7vw, 5.5rem)` in Fraunces; section headlines `clamp(1.75rem, 4vw, 3rem)`; body 1.0–1.125rem Geist with relaxed line-height (1.7); mono labels 0.75–0.8125rem.

### Logo & signature (locked brand mark — get it exactly right)
The canonical logo is the file **`inner-logo.png`** (400×400: the word "inner" in an italic serif, Bone-colored, followed by a small solid **Inner Green square sitting on the baseline as the period** — a **square, never a circle** — on an Ink background). Put it in `public/` and treat it as the single source of truth.

- Use the PNG **directly** for: the favicon, the Open Graph / social-preview image, and the Ink-background footer.
- The PNG has a solid Ink background, so it must **never** be placed on Bone sections — the black tile would clash. On Bone, reproduce the mark **typographically**, matching the PNG exactly:
  - "inner" in **Fraunces italic**, lowercase, Ink.
  - Immediately after: a solid Inner Green square resting on the baseline (~0.5–0.6em, tight optical spacing) — the period.
  - For the inner.hub lockup, append **hub** after the square: it reads `inner` `■` `hub` ("inner.hub" with the green square as the dot).
- The typographic lockup and the PNG must look identical in style, proportion, and baseline position, so the brand reads consistently across dark and light.
- In running body text, a normal period is fine; the green square is reserved for the lockup and deliberate accents.
- Optional: a very subtle opacity pulse on the square (a quiet beacon) — a 3–4s ease loop, barely noticeable. If it reads as gimmicky, make it static. Disable under `prefers-reduced-motion`.

## Layout & motion
- Editorial, left-aligned, generous negative space. Content max-width ~1120px with wide side padding (24px mobile / 64px+ desktop). Let the page breathe — spacing between sections should feel almost excessive (120–200px on desktop).
- Sections are numbered like an index, in JetBrains Mono: `01`, `02`, `03`…
- Dividers: 1px hairlines in Ink at ~12–15% opacity.
- One long scrolling page. No routes except the hidden admin view.
- Motion: content fades / slides up ~16–24px as sections enter the viewport, 400–600ms ease-out, slight stagger. Calm, never bouncy. Fully disabled under `prefers-reduced-motion`.
- Mobile-first and fully responsive; it must feel just as composed on a phone.
- Accessible: semantic HTML landmarks, real heading hierarchy, strong contrast, keyboard-navigable form, visible focus states (2px Ink outline, offset).

  - `inner.capital` — `TO FOLLOW`

### 06 — Request an invitation (the only interactive section)
- Mono label: `06 — REQUEST AN INVITATION`
- Fraunces headline:
  > Request an invitation.
- Geist sub:
  > The room is nomination-first, but good people arrive from unexpected directions. If you believe you belong here, tell us who you are. We read everything.
- Form (labels in mono, inputs styled as minimal fields: transparent background, 1px bottom hairline in Ink, no boxes, no rounded corners; focus state thickens the underline and shows the Ink outline):
  - `NAME` — text, required
  - `EMAIL` — email, required
  - `WHO YOU ARE / WHAT YOU'RE BUILDING` — textarea (3–4 rows), required
  - `LINK` — url, optional (LinkedIn or website)
  - `WHO INTRODUCED YOU` — text, optional
  - Hidden honeypot: a visually-hidden `company` input that must remain empty.
- Submit button (mono): `SEND` — Ink background, Bone text; hover inverts or underlines. No green button.
- On success, replace the form with a quiet confirmation in Fraunces italic:
  > Received. If it's a fit, we'll be in touch.
- Validation errors: calm inline messages under the field (small mono, a muted terracotta `#B4553B` — the only non-palette color, reserved strictly for errors). Never use alert boxes.

## Backend (invitation form)
- POST endpoint (e.g. `/api/request`): server-side validation (required fields, email format, URL format if provided), then store in the database with a timestamp.
- Reject any submission where the honeypot field is filled — return a generic success so bots learn nothing.
- Basic per-IP rate limiting (e.g. max 5 submissions/hour) if straightforward.
- Admin view at `/requests`: lists submissions (newest first: date, name, email, text, link, introducer), gated behind a single passcode read from the environment variable `ADMIN_PASSCODE`. A simple passcode form is enough — do not build full auth. Add `noindex` to this route.
- All secrets (passcode, database URL) live in Replit's Secrets — never hardcoded.
